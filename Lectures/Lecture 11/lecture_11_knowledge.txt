# KNN function
def knn(x0, x, y, K=3):

dist = np.abs(x - x0)
idx = np.argsort(dist)
return np.mean(y[idx[:K]])

Random forests

▶ Random forrests is an extension of bagging trees in that we,

for each bootstrap sample, we choose a subset of the
predictors. That is, we do not allow for all predictors to be
included in the trees.

▶ A consequence of this is that the predictions from the different
trees will be less correlated compared to the case in bagging.

▶ This in turn will reduce the variance of the average prediction

from the trees.

A few more words about bagged trees

▶ Bagging (bootstrap aggregation) is the idea of resampling the
training data, re-predict on the test data and finally average,
or, for classification, compute majority vote, of the predictions.

▶ The effect of averaging is to reduce the variance of the

predictions.

▶ Since averaging uncorrelated variables reduces the variance

more than averaging correlated variables, a trick to
decorrelate the predictions can be used.

▶ This method is called Random Forests and is the same as

bagged trees but with the addition that a randomly selected
subset of the predictors (X-variables) is used for each
bootstrap sample.

▶ The effect of this is a trade-off between sometimes obtaining
regression- or classification trees which exclude potentially
good predictors but, on the other hand, get less correlated
predictions.

Out-of-bag error estimation

▶ A very useful side effect with bagging is that we get a
computationally eﬀicient way of doing cross-validation.

▶ This is called out-of-bag (OUB) error estimation.

▶ We have 𝑛 observations and would like to predict 𝑌 using
bagged regression trees and the predictors 𝑋1, ..., 𝑋𝑝.
▶ In boostrap sample number 𝑏 have not sampled observation
number 𝑚. This means that we can see 𝑌𝑚 as test data for
this regression tree.

▶ As it turns out, in around one third, 𝐵/3 of the bootstrap
samples, this observation will not be present. For each of
these, we can average the predictions of 𝑌𝑚.

▶ Such test predictions can be done for each of the 𝑛

observations and consequently, e.g., a testMSE, can be
computed.

Variable importance

▶ One argument for using regression- and classification trees is
that we are able to visualize which variables that help us in
predicting the dependent variable.

▶ The argument for using bagging and Random Forests is that
we improve the predictions, which is often not very impressive
for regression- and classification trees. However, the
predictions are computed by many regression trees and it is
therefore not immediately clear which variables are
contributing to predictive accuracy.

▶ We would like to measure variable importance for the

predictors.

Variable importance

▶ In a regression tree, a measure of how important a predictor
variable is, is how much it reduces the RSS over all splits (a
variable can contribute to splits in several branches of the
tree).

▶ In bagging and Random Forests, the reduced RSS is summed

over all splits and all trees for each variable.

▶ For classification trees reduction of the Gini index is

considered rather than RSS.

▶ Boosting is similar to bagging in the sense that it is averaging

predictions over many trees.

▶ It is different in the sense that it fit the trees sequentially; in

bagging each tree is fitted to a bootstraped sample,
independently of the other trees.

▶ In boosting, the trees are sequentially fitted to a modified

version of the dataset which is updated based on the previous
trees ability to predict the different observations.
